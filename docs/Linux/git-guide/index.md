# Git Learning Guide

A comprehensive guide to understanding and using Git commands effectively.

## Documentation Structure

- [01 - Git Basics](01-basics.md) - Clone, init, and getting started
- [02 - Working with Changes](02-working-with-changes.md) - Add, commit, restore files
- [03 - History & Inspection](03-history-and-inspection.md) - View logs, diffs, and search history

- [05 - Collaboration](05-collaboration.md) - Fetch, pull, push workflows
- [Sample Workflows](examples/sample-workflows.md) - Real-world Git usage patterns

## Git Command Categories

This guide covers 5 main categories of Git commands:

1. **Start a working area** - Initialize or clone repositories
2. **Work on changes** - Stage, move, and restore files
3. **Examine history** - Review commits, differences, and status
4. **Grow and tweak history** - Branches, commits, merges, tags
5. **Collaborate** - Share changes with others
